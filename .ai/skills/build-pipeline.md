# Skill: Build Pipeline

How source files get compiled into distributable plugin assets.

## Source → Output Map

```
src/less/style.less  →  assets/css/style.css
                     →  assets/css/style.rtl.css   (auto-generated by rtlcss)
src/js/script.js     →  assets/js/script.js        (minified by Terser)
```

Git tracks both `src/` and `assets/`. The `assets/` compiled files are committed so the plugin works without a build step after install.

## Commands

```bash
npm run dev         # clean + compile + watch (LESS + JS hot rebuild on save)
npm run build       # clean + grunt + build:admin + make-pot
npm run build:admin # cd admin && npm install --include=dev && npx vite build
npm run make-pot    # regenerate languages/live-copy.pot (requires WP-CLI in PATH)
npm run zip         # build + rsync to live-copy/ folder + zip live-copy-v.X.Y.Z.zip
npm run clean       # delete assets/ and live-copy/ dirs (rimraf)
```

## Two Build Systems

| Layer | Tool | Source → Output |
|-------|------|-----------------|
| Front-end assets | Grunt | `src/less`, `src/js` → `assets/` |
| Admin React app | Vite | `admin/src` → `admin/build/{app.js,style.css}` |

`npm run build` runs Grunt then Vite. Both `assets/` and `admin/build/` are
committed so the shipped plugin runs without a build step. Admin source
(`admin/src`, `admin/node_modules`, config files) is excluded from the zip via
`.buildignore`; `admin/build/` ships.

## Grunt Tasks

Configured in `gruntfile.js`, loaded via `jit-grunt`:

### `less`
Compiles `src/less/style.less` → `assets/css/style.css`.  
Options: `compress: true`, `yuicompress: true`, `optimization: 2`.

### `rtlcss`
Reads all `assets/css/*.css` (excluding `*.rtl.css`) → generates `*.rtl.css` siblings.  
WordPress auto-loads the RTL variant when site is RTL — no PHP changes needed.

### `terser`
Minifies `src/js/script.js` → `assets/js/script.js`.  
Option: `mangle: true` (shortens variable names).

### `watch` (dev only)
- LESS files → runs `less` + `rtlcss`
- JS files → runs `terser`

## Environment Toggle

```js
// gruntfile.js
if (process.env.NODE_ENV === 'production') {
  grunt.registerTask('default', ['less', 'rtlcss', 'terser']);
} else {
  grunt.registerTask('default', ['less', 'rtlcss', 'terser', 'watch']);
}
```

`npm run dev` sets `NODE_ENV=development`, `npm run build` sets `NODE_ENV=production`.

## Zip / Release

`npm run zip` uses rsync with `.buildignore` to copy only distributable files into `./live-copy/` dir, then zips it as `live-copy-v.{version}.zip`.

Files excluded from zip (from `.buildignore`):
```
.github/    .idea/      .gitignore    .buildignore
gruntfile.js  Gruntfile.js  node_modules/  ./node_modules
.git        *.zip       .DS_Store
postcss.config.js  tailwind.config.js  webpack.config.js
src/        live-copy/  phpcs.xml     phpunit.xml    .vscode/
```

Only ships: `live-copy.php`, `includes/`, `assets/`, `languages/`, `readme.txt`.

## Dependencies

All dev dependencies — no runtime npm packages:

| Package | Purpose |
|---------|---------|
| `grunt` | Task runner |
| `grunt-contrib-less` | LESS → CSS |
| `grunt-rtlcss` | Auto RTL CSS |
| `grunt-terser` | JS minification |
| `grunt-contrib-watch` | File watcher |
| `grunt-build-number` | `buildnum` field in package.json |
| `jit-grunt` | Lazy-load grunt plugins |
| `rimraf` | Cross-platform `rm -rf` for clean task |

## Version Bump Process

1. Update version in `live-copy.php`: `LIVE_COPY_VER` constant + plugin header `Version:`
2. Update version in `package.json`: `"version"` field
3. Add changelog entry in `readme.txt`
4. Run `npm run build` to recompile
5. Run `npm run zip` to package
6. Commit + tag
